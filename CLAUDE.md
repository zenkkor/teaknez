# CLAUDE.md

Guidance for Claude Code (and any other AI coding agent) when working in this
repository.

## What this is

Static marketing site for Tea Knez Coaching (`teaknez.com`). Slovenian-language
coaching business — personal, career/business, and corporate coaching. The
site is **pure static HTML/CSS/JS**: no framework, no build pipeline beyond a
single Python file.

## How content is generated

All HTML is generated by `build.py`. It contains shared fragments (`head`,
`header`, `footer`, `testimonials_section`, `cta_band`) and per-page builder
functions (`page_index`, `page_about`, service pages, blog posts, etc.).

**To regenerate the site:**

```bash
python3 build.py
```

This writes/overwrites `*.html`, `sitemap.xml`, and `robots.txt` directly in
the project root. Do NOT hand-edit generated HTML — edit `build.py` and rerun.

## Copy is the source of truth

The Slovenian copy lives inside `build.py` (page builder functions). Do not
paraphrase or "improve" wording — copy was provided by the business owner and
should be preserved exactly unless explicitly asked to change it.

## Styling

- `assets/css/main.css` is the only stylesheet.
- Design tokens at the top: cream/clay palette, Cormorant Garamond serif +
  Inter sans.
- Body copy is **justified with hyphens** (Slovenian `lang="sl"`). Headlines,
  eyebrows, buttons, centered CTA bands, and pullquotes stay non-justified —
  see the justify block near the top of `main.css`.
- Mobile nav uses an explicit hamburger toggle at the 960px breakpoint. The
  `.site-header` deliberately drops `backdrop-filter` on mobile so the nav's
  `position: fixed` doesn't get trapped in a containing block.

## JavaScript

`assets/js/main.js` is hand-written, no bundler. It handles:

- Mobile hamburger nav (`body.nav-open`, `nav.is-open`)
- "Storitve" dropdown (hover on desktop, click-to-expand on mobile)
- Keyboard navigation (Enter/ArrowDown opens, Escape closes)
- Contact form submission to web3forms via `fetch`, with inline success/error
  messaging in `.form-status`

## Contact form

The contact page posts to `https://api.web3forms.com/submit` with a hidden
`access_key` field. Honeypot field `botcheck` is intentionally hidden via
inline style. AJAX submission is intercepted by JS — successful posts show an
inline thank-you and reset the form; failures fall back to a contact email.

## Verification

Playwright scripts under `.pw/` (gitignored `node_modules`) are used to verify
responsive layouts and form behavior. Standard viewports tested: 360, 390, 820,
1024, 1440.

```bash
cd .pw && npm install playwright   # one-time
node .pw/verify-final.mjs          # or whichever script applies
```

## What NOT to do

- Don't introduce a framework or build pipeline (React, Vite, Astro, etc.).
- Don't change the visual design without an explicit ask — the warm cream/clay
  aesthetic is intentional.
- Don't rewrite Slovenian copy.
- Don't add tracking or analytics without explicit approval.
- Don't commit `assets/` source files without optimizing first (large
  unoptimized PNGs hurt the site).

## Deploy

`main` branch → GitHub Pages, served from `/` (root). Custom domain configured
in repo Settings → Pages.
